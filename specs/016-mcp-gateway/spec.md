# Spec 016: MCP Gateway Infrastructure

## Problem

AI agents in the n8n stack need access to external tools (MLflow queries, Kubernetes management, code search, GitLab operations). Traditionally each tool server would be a persistent service in docker-compose, consuming resources even when idle. The stack has 7 MCP server images — running all constantly wastes ~3-4GB RAM.

The Docker MCP Gateway solves this by managing MCP server containers on-demand: spin up when a tool is called, spin down when idle. A single gateway endpoint multiplexes all tool servers.

## Approach

Use the official `docker/mcp-gateway` image. Tool servers are defined in `mcp-servers/catalog.yaml` (declarative). Each server is a custom Docker image built from `mcp-servers/` Dockerfiles. The gateway auto-reloads catalog changes via `--watch` flag — no restart needed when adding servers.

## Requirements

### FR-001: MCP Gateway Service

Docker Compose service `mcp-gateway` running the official gateway:

- **Image**: `docker/mcp-gateway`
- **Port**: 8811
- **Health**: `/health`
- **Docker socket**: Bind-mounted for container lifecycle management
- **Network**: `genai-net` (shared with all stack services)
- **Flags**: `--watch` (auto-reload catalog), `--catalog`, `--secrets`

### FR-002: Server Catalog

`mcp-servers/catalog.yaml` declares all available MCP servers:

| Server | Image | Purpose |
|--------|-------|---------|
| n8n-knowledge | `genai-mcp-n8n-knowledge:latest` | Node docs, template search, workflow validation |
| n8n-manager | `genai-mcp-n8n-manager:latest` | Workflow CRUD, execution management, webhook triggers |
| gitlab | `genai-mcp-gitlab:latest` | Project management, file ops, merge requests |
| mlflow | `genai-mcp-mlflow:latest` | Experiment tracking, model registry |
| kubernetes | `genai-mcp-kubernetes:latest` | Cluster management, pod ops, deployments |
| claude-code | `genai-mcp-claude-code:latest` | File ops, bash execution, code search |

Each entry specifies: `type: server`, title, description, image, env vars, secrets, and optional volume mounts.

### FR-003: Secret Management

Secrets are stored in `mcp-servers/mcp-secrets.env` (gitignored) and passed to the gateway via `--secrets` flag. Naming convention: `{server}.{secret_name}={value}`.

Generated by `task mcp:secrets` from template. Examples:
- `n8n-knowledge.n8n_api_key=...`
- `gitlab.gitlab_token=...`
- `claude-code.anthropic_api_key=...`

### FR-004: Custom Server Images

Each MCP server has a Dockerfile in `mcp-servers/`:

```
mcp-servers/
├── catalog.yaml              # server definitions
├── mcp-secrets.env           # runtime secrets (gitignored)
├── config.yaml               # machine-specific paths (gitignored)
├── config.yaml.example       # template for config.yaml
├── n8n-knowledge/
│   └── Dockerfile
├── n8n-manager/
│   └── Dockerfile
├── gitlab/
│   └── Dockerfile
├── mlflow/
│   └── Dockerfile
├── kubernetes/
│   └── Dockerfile
└── claude-code/
    └── Dockerfile
```

Build all: `task mcp:build`. Images tagged as `genai-mcp-{name}:latest`.

### FR-005: Active Server Selection

The gateway `--servers` flag controls which servers are actually loaded at startup. Not all catalog entries are activated — this allows catalog.yaml to be the full inventory while the runtime subset is controlled:

```yaml
# docker-compose.yml
command: >-
  --servers=fetch,n8n-knowledge,n8n-manager,mlflow,kubernetes,claude-code
```

**Note**: `gitlab` is defined in catalog but excluded from `--servers` due to a tool schema bug (`input schema must have type "object" (got <nil>)` panic).

### FR-006: Config.yaml Integration

Gateway configuration in `config.yaml`:

```yaml
services:
  mcp_gateway:
    port: 8811
    health: /health
    servers: "fetch,n8n-knowledge,n8n-manager,mlflow,kubernetes,claude-code"
```

### FR-007: Network Connectivity

MCP server containers join the `genai-net` Docker network via the gateway, giving them access to stack services by container name:
- `http://n8n:5678` — n8n API
- `http://mlflow:5050` — MLflow tracking
- `http://host.docker.internal:11434` — Ollama (host)

Kubernetes server requires a kubeconfig volume mount (path from `mcp-servers/config.yaml`).

### FR-008: Dashboard Integration

The Observatory Dashboard (Spec 014) monitors MCP servers by detecting containers matching `genai-mcp-*` pattern via Docker. Reports running/stopped status for each server.

### FR-009: Taskfile Automation

| Task | Purpose |
|------|---------|
| `mcp:build` | Build all MCP server Docker images |
| `mcp:secrets` | Generate mcp-secrets.env from template |
| `mcp:status` | Show gateway health + running server containers |

## Acceptance Scenarios

### SC-001: Gateway Health
Given the stack is running, `curl -sf http://localhost:8811/health` returns 200.

### SC-002: Tool Invocation
Sending an MCP tool call through the gateway starts the target server container, executes the tool, and returns the result. The container stops after idle timeout.

### SC-003: Catalog Reload
Editing `catalog.yaml` while the gateway is running causes automatic catalog reload within seconds (no gateway restart needed).

### SC-004: Server Isolation
Each MCP server container runs in isolation with only its declared env vars and secrets. A server cannot access another server's secrets.

### SC-005: Dashboard Shows MCP
The dashboard Services tab shows all 6 active MCP servers with running/stopped status.

## Non-Functional Requirements

### NFR-001: On-Demand Resource Usage
MCP server containers start on tool call and stop when idle. Only the gateway process runs persistently (~50MB RAM).

### NFR-002: Startup Latency
First tool call to a cold server incurs container startup latency (~2-5s). Subsequent calls to warm containers are sub-second.

### NFR-003: Graceful Degradation
If the MCP gateway is down, the Chat Agent still works — tool calls fail gracefully and the agent responds without tool results. MCP is an enhancer, not a blocker (Constitution Principle XI.4).

## Known Issues

1. **GitLab MCP server schema bug**: The `create_or_update_file` tool has `nil` input schema type, causing gateway panic (`AddTool: input schema must have type "object" (got <nil>)`). Excluded from `--servers` until upstream fix.
2. **Kubernetes kubeconfig**: Requires `mcp-servers/config.yaml` with local kubeconfig path. File is gitignored — must be created per-machine from `config.yaml.example`.
3. **Cold start latency**: First tool call after container eviction takes 2-5s for image pull + container start.
